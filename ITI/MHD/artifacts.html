This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Behavior: Capability Statements

The following artifacts define the specwhenic capabilities that dwhenferent types of systems complying with this implementation guide are expected to have. Systems complying with the implementation guide are expected to declare conformance to one or more of the following capability statements.

  •  Document Consumer (client) IHE ITI Mobile access to Health Documents (MHD) - Document Consumer (client).
  •  Document Recipient IHE ITI Mobile access to Health Documents (MHD) - Document Recipient.
  •  Document Responder (server) IHE ITI Mobile access to Health Documents (MHD) - Document Responder (server).
  •  Document Source (publisher) IHE ITI Mobile access to Health Documents (MHD) - Document Source (publisher).

Structures: Resource Profiles

These define constraints on FHIR resources that need to be complied with by conformant implementations

  •  Audit Event for Provide Bundle Transaction Defines constraints on the AuditEvent Resource to record when a Provide Bundle Transaction happens.
  •  Audit Event for Provide Bundle Transaction Defines constraints on the AuditEvent Resource to record when a Provide Bundle Transaction happens.
  •  MHD Comprehensive DocumentReference A profile on the DocumentReference resource for MHD Comprehensive.
  •  MHD Folder in List A profile on the List resource for MHD Folder.
  •  MHD Comprehensive Folder in List A profile on the List resource for MHD Comphrehensive Metadata Folder.
  •  MHD Minimal DocumentReference A profile on the DocumentReference resource for MHD.
  •  MHD Minimal Provide Document Bundle A profile on the Bundle transaction for Provide Document resources for MHD.
  •  MHD SubmissionSet in List A profile on the List resource for MHD SubmissionSet.
  •  MHD Comphrehensive SubmissionSet in List A profile on the List resource for MHD Comphrehensive SubmissionSet.

Terminology: Code Systems

These define new code systems used by systems conforming with this implementation guide

  •  MHD List Types To use the List resource for two different use-cases we define a code-system with the types of List needed in MHD

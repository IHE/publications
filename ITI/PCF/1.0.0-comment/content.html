3:5.8 Privacy Consent Patterns

  • 3:5.8.1 Foundation Policies
  • 3:5.8.2 Basic
  • 3:5.8.3 Intermediate
  • 3:5.8.4 Advanced

The following are the FHIR Consent profiling for the PCF profile. The FHIR Consent fundamentals are explained in Appendix P.

3:5.8.1 Foundation Policies

The Foundation Consent Content Profile indicates the common constraints for all of PCF. There are no examples of this as there is no intended use of this profile.

3:5.8.2 Basic

Using Basic Consent Content Profile

Examples for this Resource Profile:

  • Basic Consent to sharing for Treatment policy
    • ITI-71 access token
  • Basic Consent with an Ink signature
    • ITI-71 access token
  • Basic Consent that Rejects the base policy given – This could also be a representation of a situation when the patient has changed to rejection of a prior agreement
    • ITI-71 access token
  • Consent that has expired
    • ITI-71 access token

3:5.8.3 Intermediate

Using Intermediate Consent Content Profile

Examples for this Resource Profile:

  • restricted Consent to data authored in a timeframe
    • ITI-71 access token
  • restricted exception to data authored in a timeframe
    • ITI-71 access token
  • restricted consent to a named set of data
    • ITI-71 access token
  • restricted exception to a named set of data
    • ITI-71 access token
  • restricted consent to data authored by a Practitioner
    • ITI-71 access token
  • restricted exception to data authored by a Practitioner
    • ITI-71 access token
  • restricted consent to data related to an encounter
    • ITI-71 access token
  • restricted exception to data related to an encounter
    • ITI-71 access token
  • restricted consent to specific intermediate purpose of use
    • ITI-71 access token
  • Dissent except for Break-Glass.
    • ITI-71 access token

3:5.8.4 Advanced

Using Advanced Consent Content Profile

Examples for this Resource Profile:

  • Consent to allow NORMAL data access
    • ITI-71 access token
  • Consent to allow NORMAL and RESTRICTED data access
    • ITI-71 access token
  • Consent to allow NORMAL data access but not RESTRICTED – This adds the not restricted which should not be needed as the root only authorizes NORMAL
    • ITI-71 access token
  • Consent to allow NORMAL data access, and some access to RESTRICTED
    • ITI-71 access token
  • Consent to allow NORMAL data access, and some access to Mental Health
    • ITI-71 access token
  • Consent to allow NORMAL data access, and some access to Mental Health and Sexual Health
    • ITI-71 access token
  • Consent to allow NORMAL data access, and break-glass access to RESTRICTED
    • ITI-71 access token

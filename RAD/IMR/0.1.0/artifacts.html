Behavior: Capability Statements

The following artifacts define the specific capabilities that different types of systems are expected to have in order to comply with this implementation guide. Systems conforming to this implementation guide are expected to declare conformance to one or more of the following capability statements.

Terminology: Value Sets

These define sets of codes used by systems conforming to this implementation guide

IMR DiagnosticReport Status Value Set

Status codes that are applicable to imaging diagnostic report lifecycle.
IMR Study Endpoint PayloadType Value Set

Codes representing the different kinds of findings in radiology report.
IMR Read Write Update Types ValueSet

A ValueSet that allows read, write or update actions. Where the resource would typically be indicated as a GET (read), but might be created or updated depending on policy.
IMR Read Write Types ValueSet

A ValueSet that allows read or write actions. Where the resource would typically be indicated as a POST (write), but might be created or updated depending on policy.
IMR ServiceRequest intent Value Set

Codes representing the applicable intent for a ServiceRequestd.
IMR Study Endpoint ConnectionType Value Set

Codes representing the applicable endpoint connectionType to retrieve a study.
IMR Study Endpoint PayloadType Value Set

Codes representing the applicable endpoint payloadType to retrieve a study.
